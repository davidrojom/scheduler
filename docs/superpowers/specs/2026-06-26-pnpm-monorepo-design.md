# Design: pnpm monorepo refactor

**Date:** 2026-06-26
**Status:** Approved (via `/goal` directive)
**Branch:** `refactor/pnpm-monorepo`

## Problem

The repository started as a single Angular 18 app living at the repo root.
An AI-generated NestJS backend was later dropped into `backend/`, leaving an
asymmetric layout: the frontend owns the root (`src/`, `angular.json`, root
`package.json`) while the backend is a nested sub-project with its own npm
lockfile. The goal is a clean **pnpm monorepo** with two clearly separated,
peer applications: `frontend` and `backend`.

## Goals

- Single pnpm workspace with two apps as peers.
- Pure restructure: **no** dependency upgrades, **no** behavior changes, **no**
  config unification beyond what the move requires.
- Preserve git history for every moved file (`git mv`).
- One lockfile (`pnpm-lock.yaml`) replacing the two `package-lock.json` files.

## Non-Goals (YAGNI)

- No shared `packages/` (frontend and backend share no code today). The
  `apps/*` workspace glob leaves room to add one later.
- No ESLint/Prettier unification — each app keeps its own config.
- No CI pipeline (none exists today).
- No backend Docker image — the existing Dockerfile builds only the frontend
  and that stays true.

## Target Structure

```
scheduler/
├── pnpm-workspace.yaml          # packages: apps/*
├── package.json                 # NEW root: private, packageManager pin, workspace scripts
├── .npmrc                       # added ONLY if Angular needs hoisting (see Risks)
├── .gitignore                   # consolidated, covers both apps
├── .dockerignore                # stays at root, workspace-aware
├── Dockerfile                   # stays at root, build context = root, builds frontend
├── .editorconfig  README.md  CLAUDE.md   # stay at root (shared / whole-project)
├── docs/superpowers/specs/...   # this spec
├── apps/
│   ├── frontend/                # @scheduler/frontend (was "scheduler")
│   │   ├── package.json  angular.json  tsconfig*.json
│   │   ├── eslint.config.js  tailwind.config.js
│   │   └── src/  public/
│   └── backend/                 # @scheduler/backend (was "scheduler-backend")
│       ├── package.json  nest-cli.json  tsconfig*.json
│       ├── eslint.config.mjs  .prettierrc  .env.example
│       └── scripts/  src/  test/
```

## Move Plan (history-preserving)

1. `git mv` the Angular root files into `apps/frontend/`:
   `src public angular.json tsconfig.json tsconfig.app.json tsconfig.spec.json
   eslint.config.js tailwind.config.js package.json`.
2. `git mv backend apps/backend` (whole subtree).
3. Delete both `package-lock.json` and the root `node_modules` (regenerated by pnpm).

These root files **stay put**: `Dockerfile`, `.dockerignore`, `.gitignore`,
`.editorconfig`, `README.md`, `CLAUDE.md`, `.vscode/`, `.claude/`.

## New / Edited Files

- **`pnpm-workspace.yaml`** — `packages: ['apps/*']`.
- **Root `package.json`** — `private: true`, `packageManager: "pnpm@10.17.1"`,
  `engines.node >=22`, and convenience scripts:
  - `start:frontend` → `pnpm --filter @scheduler/frontend start`
  - `start:backend`  → `pnpm --filter @scheduler/backend start:dev`
  - `build` → `pnpm -r build`, `lint` → `pnpm -r lint`
- **`apps/frontend/package.json`** — rename `"scheduler"` → `"@scheduler/frontend"`.
- **`apps/backend/package.json`** — rename `"scheduler-backend"` → `"@scheduler/backend"`.
- **`angular.json`** — relative paths (`root: ""`, `src`, `public`,
  `dist/scheduler`, `tsconfig.app.json`) keep working unchanged because they
  resolve against the app dir; build output lands in `apps/frontend/dist/scheduler`.
- **`Dockerfile`** — rewrite workspace-aware: enable corepack/pnpm, copy the
  workspace, `pnpm install`, build the frontend filter, copy from
  `apps/frontend/dist/scheduler/browser` into the httpd image.
- **`.dockerignore`** — keep ignoring node_modules/dist/etc across the tree.
- **`.gitignore`** — consolidate so node_modules/dist/coverage/.env are ignored
  at any depth; drop the redundant `backend/.gitignore`.
- **`.vscode/tasks.json` + `launch.json`** — point the frontend npm tasks at
  `apps/frontend` (VS Code task `path`), keeping ng serve / ng test working.

## Risks & Mitigation

- **Angular CLI under pnpm isolated node_modules** may fail to resolve
  `@angular-devkit` builders. Mitigation: verify with a real prod build; if it
  breaks, add `.npmrc` with a targeted hoist
  (`public-hoist-pattern[]=*@angular*`, escalating to `shamefully-hoist=true`
  only if needed). This is the standard, well-documented fix.

## Verification Gate (all must pass before "done")

1. `pnpm install` completes clean.
2. `pnpm --filter @scheduler/frontend build` (production) succeeds.
3. `pnpm --filter @scheduler/backend build` (`nest build`) succeeds.
4. `pnpm -r lint` passes.
5. `pnpm --filter @scheduler/backend test` (jest) passes.
6. `git log --follow` shows preserved history on a moved file (e.g.
   `apps/frontend/src/main.ts`).
